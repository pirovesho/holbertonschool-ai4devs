# Codebase Overview - WordPress

## Age
First release in 2003, continuously updated with major releases over the years.

## Size
~500,000+ LOC (primarily PHP, with JavaScript and CSS components).

## Dependencies
- PHP
- MySQL / MariaDB
- jQuery (legacy usage in parts of the admin UI)
- Various third-party plugins and themes

## Issues
- Legacy code mixed with modern practices
- Backward compatibility constraints slow down refactoring
- Heavy reliance on plugins can introduce security risks
- Inconsistent coding standards across older components
- Performance issues at scale without optimization
