# Prompt Use Cases

## Code Quality

- **Refactoring**
  - **Goal**: Improve readability, structure, and maintainability of the code
  - **Input**: Source function or class in a programming language (e.g., Python, Java, C++)
  - **Output**: Cleaner and more organized version of the code with an explanation of improvements

- **Style Enforcement**
  - **Goal**: Ensure consistent formatting, naming conventions, and coding standards
  - **Input**: Code block or source file
  - **Output**: Reformatted code following common style guidelines (e.g., PEP8, Java conventions)


## Debugging

- **Error Diagnosis**
  - **Goal**: Identify the root cause of errors in a program
  - **Input**: Source code and the error message or unexpected behavior description
  - **Output**: Explanation of the issue and identification of the problematic code section

- **Bug Fix Suggestion**
  - **Goal**: Provide corrected code that resolves the identified issue
  - **Input**: Buggy source code
  - **Output**: Fixed version of the code and a brief explanation of the solution


## Documentation

- **Code Explanation**
  - **Goal**: Help users understand how a piece of code works
  - **Input**: Source code or function
  - **Output**: Step-by-step explanation of the code logic and behavior

- **Comment Generation**
  - **Goal**: Improve clarity by adding meaningful comments to code
  - **Input**: Source code without comments
  - **Output**: Annotated version of the code with descriptive comments


## Testing

- **Unit Test Generation**
  - **Goal**: Automatically generate tests to verify program correctness
  - **Input**: Function or module source code
  - **Output**: A set of unit tests covering typical and edge cases

- **Test Case Design**
  - **Goal**: Create meaningful test inputs to validate program behavior
  - **Input**: Description of program functionality
  - **Output**: List of test cases with expected outputs

- **Test Result Analysis**
  - **Goal**: Interpret test outcomes and identify failing cases
  - **Input**: Test execution results or logs
  - **Output**: Summary explaining which tests passed or failed and possible causes


## Learning & Code Understanding

- **Concept Explanation**
  - **Goal**: Clarify programming concepts used in code
  - **Input**: Code snippet or concept name (e.g., recursion, pointers)
  - **Output**: Simplified explanation with examples

- **Algorithm Walkthrough**
  - **Goal**: Help users understand how an algorithm works step-by-step
  - **Input**: Algorithm implementation or pseudocode
  - **Output**: Detailed explanation of the algorithm’s execution

- **Example Generation**
  - **Goal**: Provide practical examples to demonstrate code usage
  - **Input**: Function or API description
  - **Output**: Sample code showing how the function can be used in practice
